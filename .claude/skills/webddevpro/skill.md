## Instructions
- Use **React** for components and **Tailwind CSS** for styling.
- Ensure all components are responsive and accessible (WCAG AA compliant).
- Create a dedicated `/preview` route if the project supports it to showcase changes.
- Always include subtle scroll-triggered animations.

## Workflow Checklist
- [ ] Analyze existing UI patterns in the codebase.
- [ ] Draft a component plan and get user approval.
- [ ] Implement code following the project's styling conventions.
- [ ] Run a lint check and verify responsiveness.
