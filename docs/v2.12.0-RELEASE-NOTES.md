# v2.12.0 Release Notes ‚Äî Progress Media + Coach Challenges

> **Status:** In progress
> **Started:** 2026-04-13
> **Branch base:** v2.11.0

---

## Summary

v2.12.0 adds three features to the progress narrative engine:

1. **Progress Photos** ‚Äî visual evidence of change over time, attached to client snapshots with privacy-first sharing controls
2. **Session Form Check Clips** ‚Äî short video/photo capture during live sessions via inline camera, for reviewing exercise form
3. **Coach Challenges** ‚Äî measurable goals with deadlines, auto-tracked from session data, available to both trainers and athletes

- `qualitative` metric type replaces a generic "custom" label ‚Äî makes it explicit that progress is assessed by trainer/athlete judgment rather than automated measurement.

### New API routes

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/snapshots/:id/media?pose=front` | Upload progress photo (multipart). Pose required as query param. Rejects video (images only). Enforces client opt-out (403). |
| `PATCH` | `/snapshot-media/:id` | Update caption, shareable flag, or pose |
| `DELETE` | `/snapshot-media/:id` | Delete from Cloudinary + DB |
| `GET` | `/clients/:clientId/progress-photos` | All photos grouped by snapshot date (most recent first). Single query via `inArray`. Only includes snapshots that have photos. |

### Cloudinary service refactor

`uploadBuffer()` signature changed from `(buffer, exerciseId, mimeType)` to `(buffer, folder, mimeType)`. Two new folder helpers centralise the path structure:

- `exerciseFolder(exerciseId)` ‚Üí `trainer-app/exercises/<id>`
- `snapshotFolder(clientId, snapshotId)` ‚Üí `trainer-app/clients/<clientId>/snapshots/<snapshotId>`

The existing call in `media.ts` updated to use `exerciseFolder()`. No behavior change ‚Äî same Cloudinary paths as before.

### Ownership verification

All routes verify the trainer owns the resource via the snapshot ‚Üí client ‚Üí trainer chain. Two helpers encapsulate this:

- `findOwnedSnapshot(trainerId, snapshotId)` ‚Äî joins client, checks trainer ownership
- `findOwnedMedia(trainerId, mediaId)` ‚Äî joins snapshot ‚Üí client, checks trainer ownership

### Opt-out enforcement

Upload route checks `snapshot.client.progressPhotosOptedOut`. If `true`, returns `403: Client has opted out of progress photos`. The frontend should hide the capture UI, but the backend enforces it as a safety net.

### New query hooks (`lib/queries/snapshot-media.ts`)

| Hook | Purpose |
|---|---|
| `useProgressPhotos(clientId)` | Fetch progress photos grouped by snapshot date |
| `useUploadSnapshotMedia()` | Upload a progress photo (multipart FormData) |
| `useUpdateSnapshotMedia()` | Patch caption or shareable flag |
| `useDeleteSnapshotMedia()` | Delete a photo from Cloudinary + DB |

All mutations invalidate the `progress-photos` cache key on success.

### New components

**`SnapshotPhotoCapture`** ‚Äî Four camera buttons (front, side L, side R, back). Each opens the device camera via `<input type="file" capture>`. Uploaded photos replace the button with a thumbnail. Hover actions: retake and delete. When `photoSharingPreference = 'share_selected'`, each thumbnail shows a shareable toggle (eye icon). Athletes get `capture="user"` (front camera); trainers get `capture="environment"` (rear camera). Respects client opt-out ‚Äî shows disabled note when `progressPhotosOptedOut` is true. Validates image-only and 10MB cap client-side before upload.

**`ProgressPhotoTimeline`** ‚Äî Renders on the client profile Overview tab. Shows the most recent snapshot's photos in a 4-column grid. "Compare" button opens the comparison slider (only shown when 2+ snapshots have photos). Older snapshots are collapsed in a `<details>` element. Tapping any thumbnail opens the full-screen modal.

**`PhotoComparisonSlider`** ‚Äî Full-screen before/after overlay. Two date selectors (before / after) + pose chips filter which photos to compare. Draggable vertical divider reveals the "after" photo. Pointer events for smooth drag on both touch and mouse. Close via ‚úï button or Escape key.

**`ProgressPhotoModal`** ‚Äî Full-screen photo viewer. Left/right arrows to navigate between all photos. Keyboard navigation (Arrow keys + Escape). Shows pose label, date, caption, and counter.

### Integration

- `OverviewTab` now accepts `progressPhotosOptedOut` and renders `ProgressPhotoTimeline` after the training focus section and before goals.
- Both `ClientProfilePage` and `ClientProfilePanel` pass the new prop through.
- `PreferencesPage` has a new "Privacy" section at the bottom with three radio options for photo sharing preference. Saves immediately via `updatePreference()` like all other prefs.

### New Drizzle table: `session_exercise_media`

| Column | Type | Notes |
|---|---|---|
| `id` | `uuid PK` | |
| `session_exercise_id` | `uuid FK ‚Üí session_exercises (CASCADE)` | |
| `media_type` | `media_type NOT NULL` | Reuses existing pgEnum from exercises schema |
| `cloudinary_url` | `text NOT NULL` | |
| `cloudinary_public_id` | `text NOT NULL` | |
| `duration_seconds` | `integer` | Video duration ‚Äî null for images |
| `caption` | `text` | Optional note, e.g. "knee caving on rep 3" |
| `created_at` | `timestamp NOT NULL DEFAULT now()` | |

No new pgEnum needed ‚Äî reuses `media_type` (`image | video`) from exercises schema.

### New API routes

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/session-exercises/:id/media` | Upload form check clip or photo (multipart). Both images and video accepted. 30-second video cap enforced server-side ‚Äî uploads exceeding the cap are deleted from Cloudinary and rejected. |
| `DELETE` | `/session-exercise-media/:id` | Delete from Cloudinary + DB |
| `GET` | `/session-exercises/:id/media` | List all media for a session exercise, ordered by createdAt |

### Cloudinary folder

`sessionExerciseFolder(clientId, sessionId, sessionExerciseId)` ‚Üí `trainer-app/clients/<clientId>/sessions/<sessionId>/<sessionExerciseId>/`

### Ownership verification

Routes trace the full chain: `session_exercise ‚Üí workout ‚Üí session ‚Üí trainer`. Two helpers encapsulate this:

- `findOwnedSessionExercise(trainerId, sessionExerciseId)` ‚Äî joins workout ‚Üí session, checks trainer
- `findOwnedMedia(trainerId, mediaId)` ‚Äî joins sessionExercise ‚Üí workout ‚Üí session, checks trainer

### 30-second video cap

Enforced server-side as a safety net. Client sends `durationSeconds` as a query parameter (from `MediaRecorder` timer). If the value exceeds 30s, the already-uploaded Cloudinary file is deleted and a 400 is returned with a clear message. Client-side enforcement (Step 5) prevents most of these from reaching the server.

### New components

**`InlineCameraSheet`** ‚Äî Bottom sheet with live camera viewfinder via `getUserMedia`. Rear camera default (`facingMode: 'environment'`), camera flip button. Two capture modes: white button for photo (`canvas.toBlob`), red button for video (`MediaRecorder`, auto-stops at 30s). Preview screen with Retake/Use buttons before upload. Falls back to `<input type="file" capture>` when `getUserMedia` is unavailable. Cleans up media streams on close.

**`MediaPlaybackModal`** ‚Äî Full-screen viewer for form check clips and photos. Native `<video>` element with controls for video playback. Arrow key navigation between media items. Escape to close.

**`FormCheckBadge`** ‚Äî Small pill showing media count (camera icon + number). Returns null when count is 0. Used in the ExerciseBlock header during live sessions.

### Integration

**ExerciseBlock (live session):**
- Exercise header now shows a camera icon (left of the delete X) and a `FormCheckBadge` with the media count next to the exercise name
- Tapping the camera icon opens `InlineCameraSheet`
- On confirm, the captured blob is uploaded via `useUploadSessionExerciseMedia` with the session exercise ID
- Upload runs in background ‚Äî the sheet closes immediately and never blocks set logging

**SessionHistoryPanel (post-session review):**
- Each exercise card shows inline media thumbnails (up to 3, with "+N" overflow)
- Thumbnails are fetched per-exercise via `useSessionExerciseMedia` (lazy ‚Äî only when the panel is open)
- Tapping a thumbnail opens `MediaPlaybackModal` at the tapped index
- Two sub-components at the bottom of the file: `ExerciseMediaThumbs` (fetch + render thumbs) and `ExerciseMediaViewer` (fetch + render modal)
