# v2.12.0 Release Notes — Progress Media + Coach Challenges

> **Status:** In progress
> **Started:** 2026-04-13
> **Branch base:** v2.11.0

---

## Summary

v2.12.0 adds three features to the progress narrative engine:

1. **Progress Photos** — visual evidence of change over time, attached to client snapshots with privacy-first sharing controls
2. **Session Form Check Clips** — short video/photo capture during live sessions via inline camera, for reviewing exercise form
3. **Coach Challenges** — measurable goals with deadlines, auto-tracked from session data, available to both trainers and athletes

### Notes

- `photoSharingPreference` uses `.default('private')` on the response schema so existing trainer rows serialize correctly without a backfill migration.
- `progressPhotosOptedOut` uses `.default(false)` for the same reason — existing clients are opted-in by default.
- `ChallengeResponseSchema` includes a joined `exercise` field (nullable `ExerciseSummaryResponseSchema`) so the frontend can display the exercise name without a separate fetch. This follows the same pattern as `SessionExerciseResponseSchema.exercise`.
- `qualitative` metric type replaces a generic "custom" label — makes it explicit that progress is assessed by trainer/athlete judgment rather than automated measurement.

### New API routes

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/snapshots/:id/media?pose=front` | Upload progress photo (multipart). Pose required as query param. Rejects video (images only). Enforces client opt-out (403). |
| `PATCH` | `/snapshot-media/:id` | Update caption, shareable flag, or pose |
| `DELETE` | `/snapshot-media/:id` | Delete from Cloudinary + DB |
| `GET` | `/clients/:clientId/progress-photos` | All photos grouped by snapshot date (most recent first). Single query via `inArray`. Only includes snapshots that have photos. |

### Cloudinary service refactor

`uploadBuffer()` signature changed from `(buffer, exerciseId, mimeType)` to `(buffer, folder, mimeType)`. Two new folder helpers centralise the path structure:

- `exerciseFolder(exerciseId)` → `trainer-app/exercises/<id>`
- `snapshotFolder(clientId, snapshotId)` → `trainer-app/clients/<clientId>/snapshots/<snapshotId>`

The existing call in `media.ts` updated to use `exerciseFolder()`. No behavior change — same Cloudinary paths as before.

### Ownership verification

All routes verify the trainer owns the resource via the snapshot → client → trainer chain. Two helpers encapsulate this:

- `findOwnedSnapshot(trainerId, snapshotId)` — joins client, checks trainer ownership
- `findOwnedMedia(trainerId, mediaId)` — joins snapshot → client, checks trainer ownership

### Opt-out enforcement

Upload route checks `snapshot.client.progressPhotosOptedOut`. If `true`, returns `403: Client has opted out of progress photos`. The frontend should hide the capture UI, but the backend enforces it as a safety net.

### New query hooks (`lib/queries/snapshot-media.ts`)

| Hook | Purpose |
|---|---|
| `useProgressPhotos(clientId)` | Fetch progress photos grouped by snapshot date |
| `useUploadSnapshotMedia()` | Upload a progress photo (multipart FormData) |
| `useUpdateSnapshotMedia()` | Patch caption or shareable flag |
| `useDeleteSnapshotMedia()` | Delete a photo from Cloudinary + DB |

All mutations invalidate the `progress-photos` cache key on success.

### New components

**`SnapshotPhotoCapture`** — Four camera buttons (front, side L, side R, back). Each opens the device camera via `<input type="file" capture>`. Uploaded photos replace the button with a thumbnail. Hover actions: retake and delete. When `photoSharingPreference = 'share_selected'`, each thumbnail shows a shareable toggle (eye icon). Athletes get `capture="user"` (front camera); trainers get `capture="environment"` (rear camera). Respects client opt-out — shows disabled note when `progressPhotosOptedOut` is true. Validates image-only and 10MB cap client-side before upload.

**`ProgressPhotoTimeline`** — Renders on the client profile Overview tab. Shows the most recent snapshot's photos in a 4-column grid. "Compare" button opens the comparison slider (only shown when 2+ snapshots have photos). Older snapshots are collapsed in a `<details>` element. Tapping any thumbnail opens the full-screen modal.

**`PhotoComparisonSlider`** — Full-screen before/after overlay. Two date selectors (before / after) + pose chips filter which photos to compare. Draggable vertical divider reveals the "after" photo. Pointer events for smooth drag on both touch and mouse. Close via ✕ button or Escape key.

**`ProgressPhotoModal`** — Full-screen photo viewer. Left/right arrows to navigate between all photos. Keyboard navigation (Arrow keys + Escape). Shows pose label, date, caption, and counter.

### Integration

- `OverviewTab` now accepts `progressPhotosOptedOut` and renders `ProgressPhotoTimeline` after the training focus section and before goals.
- Both `ClientProfilePage` and `ClientProfilePanel` pass the new prop through.
- `PreferencesPage` has a new "Privacy" section at the bottom with three radio options for photo sharing preference. Saves immediately via `updatePreference()` like all other prefs.
