# v2.12.0 Release Notes — Progress Media + Coach Challenges

> **Status:** In progress
> **Started:** 2026-04-13
> **Branch base:** v2.11.0

---

## Summary

v2.12.0 adds three features to the progress narrative engine:

1. **Progress Photos** — visual evidence of change over time, attached to client snapshots with privacy-first sharing controls
2. **Session Form Check Clips** — short video/photo capture during live sessions via inline camera, for reviewing exercise form
3. **Coach Challenges** — measurable goals with deadlines, auto-tracked from session data, available to both trainers and athletes

### Notes

- `photoSharingPreference` uses `.default('private')` on the response schema so existing trainer rows serialize correctly without a backfill migration.
- `progressPhotosOptedOut` uses `.default(false)` for the same reason — existing clients are opted-in by default.
- `ChallengeResponseSchema` includes a joined `exercise` field (nullable `ExerciseSummaryResponseSchema`) so the frontend can display the exercise name without a separate fetch. This follows the same pattern as `SessionExerciseResponseSchema.exercise`.
- `qualitative` metric type replaces a generic "custom" label — makes it explicit that progress is assessed by trainer/athlete judgment rather than automated measurement.
